## Installation

```bash
pip install agent-auth
```

<Note>
  Parameters in [square brackets] are optional. All other parameters are required.
</Note>

## Global configuration

The CLI can be configured in two ways (in order of precedence):
1. Command line arguments
2. Environment variables

### Environment variables

```bash
AUTHED_REGISTRY_URL="https://api.getauthed.dev"
AUTHED_PROVIDER_ID="<your-provider-id>"
AUTHED_PROVIDER_SECRET="<your-provider-secret>"
```

## Commands

### Initialize configuration

Configure CLI:
```bash
agent-auth init config
```

Show current config:
```bash
agent-auth init show
```

Clear config (--force skips confirmation):
```bash
agent-auth init clear [--force]
```

### Agent management

List agents:
```bash
agent-auth agents list [--output <FILE>]
```

Create agent:
```bash
agent-auth agents create --name <AGENT_NAME> [--key-file <KEY_FILE>]
```

Get agent info:
```bash
agent-auth agents info <AGENT_ID>
```

Delete agent (--force skips confirmation):
```bash
agent-auth agents delete <AGENT_ID> [--force]
```

### Key management

Generate keypair:
```bash
agent-auth keys generate [--output <FILE>]
```

Show public key:
```bash
agent-auth keys show-public <KEY_FILE>
```

Show private key (--force required as safety measure):
```bash
agent-auth keys show-private <KEY_FILE> --force
```

### Permission management

List permissions:
```bash
agent-auth permissions list <AGENT_ID> [--output <FILE>]
```

Grant agent permission:
```bash
agent-auth permissions grant <AGENT_ID> \
  --type allow_agent \
  --target-id <TARGET_AGENT_ID>
```

Grant provider permission:
```bash
agent-auth permissions grant <AGENT_ID> \
  --type allow_provider \
  --target-id <TARGET_PROVIDER_ID>
```

Revoke permission (--force skips confirmation):
```bash
agent-auth permissions revoke <AGENT_ID> \
  --type allow_agent \
  --target-id <TARGET_ID> \
  [--force]
```

Clear permissions (--force skips confirmation):
```bash
agent-auth permissions clear <AGENT_ID> [--force]
```

### Log management

Stream logs:
```bash
agent-auth logs stream \
  --agent-id <AGENT_ID> \
  [--level INFO|WARNING|ERROR] \
  [--output <FILE>]
```

Fetch logs:
```bash
agent-auth logs fetch \
  --agent-id <AGENT_ID> \
  [--level INFO|WARNING|ERROR] \
  [--limit <NUMBER>] \
  [--output <FILE>]
```

## Common workflows

### Setting up a new agent

1. Generate keys:
```bash
agent-auth keys generate --output agent_keys.json
```

2. Create agent:
```bash
agent-auth agents create \
  --name "my-agent" \
  --key-file agent_keys.json
```

3. Verify creation:
```bash
agent-auth agents info <AGENT_ID>
```

### Managing permissions

List current permissions:
```bash
agent-auth permissions list <AGENT_ID>
```

Grant agent access:
```bash
agent-auth permissions grant <AGENT_ID> \
  --type allow_agent \
  --target-id <TARGET_AGENT_ID>
```

Grant provider access:
```bash
agent-auth permissions grant <AGENT_ID> \
  --type allow_provider \
  --target-id <TARGET_PROVIDER_ID>
```

## Output formats

### JSON output
When using `--output` flag, data is saved in JSON format:

```json
{
  "agents": [
    {
      "agent_id": "...",
      "name": "...",
      "provider_id": "...",
      "created_at": "...",
      "permissions": [
        {
          "type": "allow_agent",
          "target_id": "..."
        }
      ]
    }
  ]
}
```

### Terminal output
Without `--output`, the CLI provides colored, formatted output:
- Success messages in green (✓)
- Warnings in yellow (⚠️)
- Errors in red (✗)
- IDs and technical values in different colors for readability
